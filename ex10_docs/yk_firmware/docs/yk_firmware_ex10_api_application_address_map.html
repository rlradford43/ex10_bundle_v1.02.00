InterruptMask: The interrupt enable mask¶

This register sets the value for the interrupt mask which determines which interrupt sources will trigger an interrupt on the IRQ_N pin.

  • 1: indicates that the specific interrupt is enabled.

  • 0: indicates that the specific interrupt is disabled.

Field Address: 0x00A0

Field Details:

Bit

Name

Description
0 OpDone

The most recently executed operation has completed. Details on executed operation can be retrieved from the OpsStatus register.
1 Halted

The modem has halted on a tag and is now awaiting halted instructions from the host. This interrupt will be set by the firmware once upon entering halted state. HaltedControl register may now be used to control the modem. The host now may update the Gen2TxBuffer, Gen2TxnControls, Gen2TransactionIds, Gen2Lengths, Gen2Offsets and Gen2AccessEnable registers to configure the desired Access sequence.

These Gen2-related registers may be pre-loaded with the desired Access sequence ahead of starting an inventory round. Setting Go field in HaltedControl register will initiate enabled Gen2 sequences defined in the appropriate Gen2 registers.
2 EventFifoAboveThresh

The EventFifo byte count is over the threshold defined in the EventFifoIntLevel register. The available byte count of the data in the EventFifo may be read from the EventFifoNumBytes register and the host should use the ReadFifo command to retrieve the data.

For more details, see Impinj Reader Chip SDK Event Packet Processing.
3 EventFifoFull

The EventFifo is full and the modem is now stalled until the host reads some of the EventFifo data. The host should use the ReadFifo command to retrieve the data.

For more details, see Impinj Reader Chip SDK Event Packet Processing.
4 InventoryRoundDone

This interrupt will be set at the completion of every inventory round initiated by StartInventoryRoundOp.
5 HaltedSequenceDone

The modem has completed an access sequence on a tag and is now awaiting further halted instructions from the host. Starting an access sequence is triggered by setting Go field in HaltedControl register. While in halted state, the modem will set this interrupt each time it completes an Access sequence.
6 CommandError

A response-less command (command that does not return a response has completed with a failing ResponseCode. Any completion code other than ‘Success’ will trigger this interrupt. The CommandResult register will hold further details about the error that occurred.

The ETSI burst operation is used during testing to repeatedly ramp up and ramp down the transmitter with the applied modulation so that the demodulated waveform can be measured. OpsStatus: Operation status register¶

This register reflects the current operation status. When an operation starts, this register will indicate the operation ID and the ‘Busy’ field will be set, and the ‘Error’ field will be set to ‘None’. Once an operation completes, it will clear the ‘Busy’ field in this register and update the ‘Error’ field if any errors were encountered. The OpId field will reflect either the currently running operation ID, or the last operation ID (until another operation is run).

CalibrationInfo: Stored calibration data¶

This register provides read-only access to the calibration data stored in the Impinj reader chip calibration info page. This flash page is separate from the memory used by the firmware image and persists across firmware updates.

Calibration data can be used to tune the performance of the reader output relative to input from internal sensors, such as power output sensors and temperature sensors, for example, so that the output characteristics are more precise. This can be especially important for meeting regulatory requirements.

The format of the data in this area is application specicific, and it can can contain any data in any format that supports reader calibration, as long as it is 2KB or less.

The host writes the calibration data into the flash info page from the bootloader. There are funtions for writing to this flash page from all three layers of the Impinj Reader Chip Host API.

  • Ex10Reader layer

    • write_cal_info_page()

  • Ex10Ops layer

    • write_calibration_page() (Ex10Ops)

  • Ex10Protocol layer

    • write_calibration_page() (Ex10Protocol)

    • erase_calibration_page()
