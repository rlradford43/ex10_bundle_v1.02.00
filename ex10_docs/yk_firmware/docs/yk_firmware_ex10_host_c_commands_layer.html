Ex10Commands Layer¶

The Impinj reader chip commands provide a thread-safe C language functional interface to the Impinj reader chip over the wireline communication interface.

struct Ex10Commands¶

enum ResponseCode (*read)(struct RegisterInfo const *const address_lengths[], struct ByteSpan *byte_spans[], size_t number_of_spans, uint32_t ready_n_timeout_ms)¶

Read one or more byte spans of memory on the device.

Given an addresses and lengths to read, access the device over SPI bus to read the byte spans.

Note

If response is not Success, outputs error information to stderr and asserts failure.

Parameters

  • address_lengths – A pointer to a buffer of register_info structs, from hich the address and length are used.

  • byte_spans – A pointer to an array of buffer pointers to return data read from the Ex10 device.

  • number_of_spans – The number byte_spans to read.

  • ready_n_timeout_ms – The number of milliseconds to wait for the Ex10 READY_N line to assert low.

Returns

enum ResponseCode and updated byte_spans

enum ResponseCode (*read_fifo)(enum FifoSelection selection, struct ByteSpan *byte_span)¶

Read the specified number of bytes from device FIFO stream.

Given a selected fifo, read bytes from the fifo into a ByteSpan.

Multiple ReadFifo commands are made until the byte_span length value is filled to capacity.

The byte_span->length should be the number of bytes obtained by reading the EventFifoNumBytes register. It does not include the response code byte.

The byte_span->length value will be updated with the number of bytes read using the ReadFifo command into the buffer.

Note

If 32-bit alignment of the event fifo messages is required then byte_span->data[1] must be u32 aligned since data[0] will contain the response code.

Note

If response is not Success, outputs error information to stderr and calls asserts failure.

Warning

If the byte_span value is greater than the number of bytes contained in the fifo then this function will continue to read from the fifo until the length value is satisfied. Therefore the byte_span length value should contain the value read from EventFifoNumBytes register.

Parameters

  • selection – Which FIFO to read from. In the Ex10 only the EventFifo (0) is supported.

  • byte_span – A struct with a pointer to a ByteSpan struct to hold the requested bytes, along with a length.

Returns Success

Indicates the command was successful.

Returns

ResponseCode

enum ResponseCode (*test_transfer)(struct ConstByteSpan const *send, struct ByteSpan *recv, bool verify)¶

Run the Transfer Test command and get the response.

Note

If response is not Success, or received data content or length is not as expected, outputs error information to stderr and asserts.

Parameters

  • send – A ConstByteSpan to send as the transfer test payload.

  • recv – A ByteSpan of received bytes from the transfer test. ByteSpan length must be => length of the ByteSpan to send.

  • verify – Check the test response and return error if incorrect.

Returns

ResponseCode
