Ex10Reader Layer¶

The Ex10Reader interface provides the highest level of abstraction, yet is still a full featured interface for performing RAIN RFID tasks. The Impinj Reader Chip Host API is described in this section.

struct OpCompletionStatus (*inventory)(uint8_t antenna, uint16_t rf_mode, uint16_t tx_power_dbm, struct InventoryRoundControlFields const *inventory_config, struct InventoryRoundControl_2Fields const *inventory_config_2, bool send_selects, uint32_t frequency_khz, bool remain_on)¶

Run a single inventory round.

Note

100 * decibel => millibel

Note

left out dual target on purpose - should be handled by calling code based on inventory parameters

Parameters

  • antenna – The antenna to transmit on.

  • rf_mode – An Ex10 RF Mode to be used for this round.

  • tx_power_dbm – Target transmit power level in 100th’s of a dbm. Example: 2,950 = 29.5 dBm

  • inventory_config – The inventory settings to be used for this round.

  • inventory_config_2 – The second inventory settings register

  • select_config – The parameters used for a select sent before each inventory round. By passing in a select configuration through the inventory call, you are overriding any previous command specifications made in the TX buffer.

  • frequency_khz – If a frequency is specified here, this forces the stack to always ramp up to the same frequency. If 0, the region specific jump table will be used.

  • remain_on – If false, the normal region specific regulatory timers will be used. If true, the timers will be disabled and the Ex10 device will stay powered on until told to ramp down.

Returns

Info about any encountered errors.

void (*packet_remove)(void)¶

Delete the packet at the front of the packet queue.

Note

There is a limited amount of room for packets in the packet queue, so packets should be discarded regularly using remove_packet() to ensure room for new packets.

Note

The packet must only be removed once it is not longer in use. packet_remove() is similar to calling free() on allocated memory. Once the packet has been removed its packet pointer obtained through packet_peek() is invalid.

Note

Calling packet_free() allows for new packets to be read from the Ex10. If packet_remove() is not called then reading will stall and the available buffer space will have been used up.

bool (*packets_available)(void)¶

Query whether packets are available for reading.

Note

This is merely a proxy for calling packet_peek() and returns true if packet_peek() returns non-NULL.

Returns

bool true if packets are available for reading. false if there are no packets available for reading.
