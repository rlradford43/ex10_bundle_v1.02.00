Impinj Reader Chip Bootloader Address Map¶

These are the registers that can be accessed with the Read and Write commands when the Impinj reader chip is in the bootloader context. This section lists the registers specific to the bootloader and are in addition to the common registers listed in the Impinj Reader Chip Common Address Map section. When fields are multi-byte, they are little-endian.

FrefFreq: FREF frequency [kHz]¶

Only the values of 19200, 24000, 38400, and 57600 are supported for bootloader operation. The enumeration value of 0 will prevent the bootloader from programming the flash. Setting this value also adjusts the timing for the flash controller.

RemainReason: Reason for remaining in bootloader¶

Indicates the reason why the bootloader is executing and the application was not called from the bootloader.

Field Address: 0x0038

Field Details:

Bit

Name

Description
7:0 RemainReason

Value

Name

Description
0x0 NoReason

Reading this value within the bootloader indicates an unexpected error.
0x1 ReadyNAsserted

The READY_N input pin was asserted (low) during reset.
0x2 ApplicationImageInvalid

No valid firmware application image was present. If an application firmware upload was aborted before completing, this reason would be given.
0x3 ResetCommand

The reset command was issued and the parameter ‘Destination’ was set to ‘Bootloader’.
0x4 Crash

A crash was encountered and the stored setting indicated that the bootloader should not jump to the application.
0x5 Watchdog

The watchdog timer expired and the stored setting indicated that the bootloader should not jump to the application.
0x6 Lockup

A lockup condition was encountered and the stored setting indicated that the bootloader should not jump to the application.

ImageValidity: Application image validity markers¶

Field Address: 0x0039

Field Details:

Bit

Name

Description
0 ImageValidMarker

0 - Indicates that the main flash image has not been marked valid 1 - Indicates that the main flash image has been marked valid
1 ImageNonValidMarker

0 - Indicates that the main flash image has not been marked invalid 1 - Indicates that the main flash image has been marked invalid

BootloaderVersionString: The bootloader version string¶

The version string shall conform with Semantic Versioning 2.x https://semver.org/

BootloaderBuildNumber: The bootloader build number¶

The build number assigned by the CI system when building (and testing) the bootloader ROM firmware.

BootloaderGitHash: The bootloader git hash¶

The Git (version control tool) SHA that identifies the state of the source code used to build the bootloader ROM firmware.

CrashInfo: Crash info memory¶

Reads low level details of an application firmware crash. This may help identify the cause of a crash. Impinj internal use only.
