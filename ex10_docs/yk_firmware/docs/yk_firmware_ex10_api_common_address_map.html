Impinj Reader Chip Common Address Map¶

These are the registers that can be accessed with the Read and Write commands from the both the bootloader and application contexts. When fields are multi-byte, they are little-endian.

CommandResult: Command message result for responseless commands¶

Commands which provide a response contain the command result code within the response. Commands which do not have a response and fail to perform the requested command store the failed command and response codes in these registers.

These values are cleared to indicate ‘no error’ once any portion of the command result register set is read. The FailedResultCode field is set to its ‘success’ value 0xA5. The FailedCommandCode and CommandsSinceFirstError fields are reset to the value of 0x00.

Field Address: 0x0000

Field Details:

Bit

Name

Description
7:0 FailedResultCode

The failed result code of the first command which failed to perform its intended function and contains no response. Subsequent commands which have no response and fail do not overwrite this value until this register is read. When this register is read all fields within the register are reset to their default values.

31:16 CommandsSinceFirstError

The number of commands that were received over the host interface that did not provide a response since the first command failure occurred. This value indicates the staleness of the FailedResultCode and FailedCommandCode values.

ResetCause: The cause of the reader chip reset.¶

The bitfields indicate the various possible reasons for the reset. When none of the fields are set, that indicates the most recent reset was due to a POR.

Field Address: 0x0004

Field Details:

Bit

Name

Description
0 SoftwareReset

The firmware initiated the reset.
1 WatchdogTimeout

2 Lockup

3 ExternalReset

VersionString: The application firmware version string¶

The version string conforms with Semantic Versioning 2.x https://semver.org/

BuildNumber: The application firmware build number¶

The build number assigned by the CI system when building (and testing) the application firmware.

GitHash: The application firmware git hash¶

The Git (version control tool) SHA that identifies the state of the source code used to build the application firmware. Only the 4 left most bytes are used, in byte order.

SerialNumber: The device serial number¶

The serial number uniquely identifies the Impinj reader chip amongst all other devices which have been manufactured. Each device is assigned a unique 32 byte value.
