Ex10Protocol Layer¶

The Ex10Protocol interface provides an abstraction of the wireline protocol. The Ex10Ops object uses this interface to abstract the sequence of register writes/reads necessary to implement the various operations.

struct Ex10Protocol¶

Ex10 Protocol interface.

Public Members

void (*init)(struct Ex10DriverList const *driver_list)¶

Initialize the Ex10Protocol object.

This sets up access to an Impinj Reader Chip.

Note

This function does not perform any interaction with the Impinj Reader Chip; it only sets up the object.

Param driver_list

The interface to use to communicate with the Ex10.

int (*init_ex10)(void)¶

Initialize the Impinj Reader Chip at the Ex10Protocol level.

  • Sets the default EventFifo threshold level to 2048 bytes.

  • Clears the Ex10 interrupt mask register.

  • Clears the interrupt status register by reading it.

Note

The Ex10 must be powered on prior to calling this function. The chain of Ex10 module processing is enabled in this call.

Returns 0

Indicates the initialization was successful.

Returns !=

0 Indicates a pthread related initialization error.

Return

int A POSIX error code indicating whether board initialization passed or failed.

int (*reset)(enum Status destination)¶

Issue a soft reset to the Ex10 chip.

If there is no valid application image loaded into the Ex10, then the request to reset into the Application will result in a reset into the Bootloader.

Note

Requesting a reset into the Application does not guarantee that the device will actually enter Application mode.

Param destination

The requested execution context of the Ex10 device: Application or Bootlodaer.

Returns =

0 The reset to the requested destination was successful.

Returns =

1 The Impinj Reader Chip reset to the Bootloader when the Application was requested.

Returns <

0 An operating system error code: The negative POSIX errno value.

Return

int A integer error code indicating success or failure.
