Advanced Inventory Features¶

The Impinj Ex10 Reader Chip has many advanced features that can be used to control how it conducts an inventory round. Listed below are a few of the features and how to use them.

Drop Query¶

The drop query feature is a feature that will lower the transmit power for the initial Query command that starts an inventory round. This feature will work with any of the existing inventory round features and an example use case is described below. There are two parameters that control this feature and they are located in the DropQueryControl Ex10 register. The first field is the DropPower field which sets the number of cdB to drop the power from the existing power level. This field has a range from 0 to 200 where 0 the feature is disabled, and at 200 the power will be dropped by 2 dB. The power drop is limited to 200 cdB so if the user enters a value greater than 200, a value of 200 cdB will be used. The second field is the DropDwell field that controls the amount of time to dwell at that lower power state before sending the Query command. If the DropPower field is set to 0 it will disable the feature. So if DropPower is 0 but DropDwell is not the feature will still be disabled (to dwell time will be observed). These registers default to 0 at reset which means that this feature is disabled by default.

Note that the drop query feature has only been verified on FM0 modes, which are modes 11, 102, 103, 202, and 302. It is not recommended to use this feature on any other modes.

Below is an illustration of the drop query feature. It starts with the transmitter transmitting CW at the power the device was ramped to. Then it drops the power of the transmitter and dwells there, then emits the Query command and restores the transmitter’s output to the original power for the RN16 to be received. Then it continues on with the inventory round as normal.

This feature can be useful in environments where there is a significant portion of the population of tags in the field of view that are on the edge of sensitivity. This is especially true for tags that are on the edge of the TX sensitivity because they will try to participate in the inventory round, but can’t complete the transaction and move to the other target state. Since they do not move to the other target state they will try to participate and collide with other tags after every QueryAdj and cause the LMAC to continue to try to read them until it has not seen a valid EPC after the defined number of queries.

The drop query feature then allows the user to lower the power for just the Query portion of the inventory round, so those fringe tags will not see the original Query and will not participate in the inventory round. This allows the LMAC to move all of the stronger tags over to the other target and stop quicker. This method can be espeically useful if the host wants to use a fast mode to move the easy to read tags over to the other target, then switch to a more sensitive mode for the harder to read tags.
