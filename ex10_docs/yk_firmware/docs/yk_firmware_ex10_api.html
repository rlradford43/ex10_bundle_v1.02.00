Impinj Reader Chip Wireline API¶

The Impinj RAIN RFID reader chips (“Impinj reader chips”) have an integrated Cortex-M0+ microcontroller with embedded firmware that implements the GS1 UHF Gen2 RAIN protocol. A host processor communicates with the Impinj reader chip via Serial Peripheral Interface (SPI). The Impinj reader chip wireline API provides a low-level interface across the entire Impinj reader chip family of products including the Impinj E910, E710, E510 and E310. The Impinj reader chip wireline API is comprised of the 5 components listed below and shown in Fig. 6:

Wireline

Message framing and flow control

Messages

The set of messages transported over the wireline

Address Map

Most of the configuration and status is accessed in this address map via the Read and Write messages

Operations

These are the actions that can be done with the Impinj reader chip. They are initiated via writes to the address map and will generate interrupts to the host when complete. Some operations will produce data in the EventFifo.

FIFOs

Some data are streamed back to the user in FIFOs, for example tag reports.
