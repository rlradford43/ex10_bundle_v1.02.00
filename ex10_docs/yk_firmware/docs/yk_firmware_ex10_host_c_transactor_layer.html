Ex10Transactor Layer¶

There are two interface modules which reside within the Ex10Transactor layer: Ex10CommandTransactor and Ex10PowerTransactor.

The Ex10CommandTransactor uses the HostInterface to send and recieve data across the wireline to and from the Impinj Reader Chip.

The Ex10PowerTransactor manipulates the host GPIO lines to power up/down the Impinj Reader Chip into the Bootloader and/or Application.

struct Ex10CommandTransactor¶

Public Members

void (*init)(struct Ex10GpioInterface const *gpio_interface, struct HostInterface const *host_interface)¶

Initialize the command transactor object.

This resets the command transactor to the command state. The host interface and gpio modules are opened and owned by the command transactor.

void (*deinit)(void)¶

Release any resources in use by the command transactor module.

This includes the host interface. The host interface and gpio modules are closed and released from ownership by the command transactor.

void (*send_command)(const void *command_buffer, size_t command_length, uint32_t ready_n_timeout_ms)¶

Sends a command from the host to the Ex10.

Blocks until READY_N is asserted by Ex10.

Asserts failure when:

  • The gpio and wireline interfaces are not initialized.

  • The Ex10 READY_N line does not assert low before the timeout expires.

  • The number of bytes sent does not equal the number of bytes requested over the wireline interface.

Param command_buffer

Contains the Ex10 command.

Param command_length

The length of the command in bytes.

Param ready_n_timeout_ms

The number of milliseconds to wait for the Ex10 READY_N line to assert low.

size_t (*receive_response)(void *response_buffer, size_t response_buffer_length, uint32_t ready_n_timeout_ms)¶

Receives a response from the host.

Blocks waiting for READY_N to be asserted by the Ex10.

Asserts failure when:

  • The gpio and wireline interfaces are not initialized.

  • The Ex10 READY_N line does not assert low before the timeout expires.

Param response_buffer

The buffer which will be filled in with the Ex10 response.

Param response_buffer_length

The size of the response buffer in bytes.

Param ready_n_timeout_ms

The number of milliseconds to wait for the Ex10 READY_N line to assert low.

Return

size_t The number of bytes received in the response buffer.

size_t (*send_and_recv_bytes)(const void *command_buffer, size_t command_length, void *response_buffer, size_t response_buffer_length, uint32_t ready_n_timeout_ms)¶

Calls send_command() and receive_response() with one function.
